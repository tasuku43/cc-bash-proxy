# Quickstart

`cc-bash-guard` evaluates Bash commands against permission policy and returns
`allow`, `ask`, or `deny` for Claude Code hooks. It does not rewrite commands.

## Install And Initialize

Create the user config and print the Claude Code hook snippet:

```sh
cc-bash-guard init
```

The user config lives at:

```text
~/.config/cc-bash-guard/cc-bash-guard.yml
```

A project can also add an optional local config:

```text
.cc-bash-guard/cc-bash-guard.yaml
```

## Minimal Policy

Start with a small policy and verify it before relying on the hook.

```yaml
permission:
  deny:
    - name: git force push
      command:
        name: git
        semantic:
          verb: push
          force: true

  allow:
    - name: git read-only
      command:
        name: git
        semantic:
          verb_in:
            - status
            - diff
            - log
            - show

    - name: read-only shell basics
      patterns:
        - "^ls(\\s|$)"
        - "^pwd$"
```

## Verify

Run verify after editing policy:

```sh
cc-bash-guard verify
```

Use doctor for installation and configuration diagnostics:

```sh
cc-bash-guard doctor
```

## Hook Setup

Configure Claude Code to call `cc-bash-guard hook` for Bash `PreToolUse`.
`cc-bash-guard init` prints the snippet for your environment.

The hook uses verified artifacts generated by `cc-bash-guard verify`. Without
`--auto-verify`, the hook fails closed when the artifact is missing or stale.
